# Todo List Application (Java Console)

## Описание проекта
Консольное Java-приложение для управления списком задач. В проекте реализован чистый ООП-подход с разделением ответственности между слоями (Separation of Concerns).

Технологический стек:
* Java 8+ (использованы Stream API, Lambda).
* File I/O для хранения данных.
* UUID для уникальной идентификации задач.

## Функционал 
* Добавление задач: Каждая задача получает уникальный ID (UUID) и статус `NEW` по умолчанию.
* Просмотр: Вывод списка всех задач в отформатированном виде.
* Управление статусом: Смена статуса задачи на `NEW`, `IN_PROGRESS` или `DONE`.
* Удаление: Удаление по ID.
* Автосохранение: Данные сохраняются в файл `tasks.txt` после каждой операции.
* Загрузка: При запуске программа автоматически считывает существующие задачи.

## Архитектура 
Приложение построено по многослойной структуре:
1. `model` — описание сущности `Task` и перечисления `Status`.
2. `repository` — интерфейс и реализация работы с данными (Data Access Layer).
3. `service` — бизнес-логика приложения (`TaskManager`).
4. `ui` — взаимодействие с пользователем через консоль.

## 🛠 Инструкция по запуску [
1. Скачайте и распакуйте архив.
2. Откройте проект в IntelliJ IDEA (или любой другой IDE).
3. Скомпилируйте файлы и запустите класс `Main`.
4. Следуйте инструкциям в консольном меню.

## Что бы я улучшил дальше
1. Переход на JSON/БД: Заменить хранение в txt на более структурированный формат (например, JSON с использованием Jackson/Gson) или базу данных (PostgreSQL/H2).
2. Обработка ошибок: Добавить более глубокую валидацию ввода (например, проверка формата при добавлении задачи или защита от неверных символов в меню).
3. Unit-тестирование: Покрыть бизнес-логику (папка service) юнит-тестами с помощью JUnit и Mockito.
4. Улучшение архитектуры: Применить внедрение зависимостей (Dependency Injection) или использовать фреймворк, такой как Spring Boot, при дальнейшем росте приложения.
